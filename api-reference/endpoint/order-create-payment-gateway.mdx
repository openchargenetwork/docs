---
title: "Create Order (Payment Gateway)"
description: "Payment gateways implement this endpoint to receive signed orders from merchants via QR code scanning. The merchant scans the user's QR code (containing the session URL) and posts their signed order."
openapi: "POST /orders/create/{sessionId}"
---

<Info>
For detailed flow diagrams, QR code format, and implementation guidance, see the [Merchant Orders guide](/guides/protocol/merchant-orders#payment-gateway-flow).
</Info>

**Implements:** Payment gateways

**Called by:** Merchants

**Path parameter:** `sessionId` - Temporary session ID from the QR code. Single-use and expires after timeout.
